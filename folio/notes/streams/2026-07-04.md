# Building a MUD in Sigil

Building a MUD from scratch, live: the classic multiplayer text world, reimagined with a modern web interface and written entirely in Sigil.

## News

- **First real build stream.** The relaunch ([[streams/2026-06-26]]) was about the why and the how. This one is about actually making something, live, from an empty file.
- **Sigil Web gets its first real workout!** The game's interface runs on Sigil Web, the framework I've been building for live, in-the-browser interfaces.

## The Plan

The goal for today: a MUD you can actually move around in by the end, running in the browser, driven live by the game as it runs.

- **What we're making.** A text-based MUD, the classic type-to-play multiplayer world, but with a modern web interface instead of a bare terminal. Rooms you can walk between, commands you can type, and a world that updates in real time.
- **How the interface works.** The front end runs on Sigil Web. It streams HTML fragments over SSE, HTMX-style, so the page updates and responds as the game runs, without a pile of client-side JavaScript.
- **The real experiment.** I've spent a long time building Sigil the language. Today I want to find out what it feels like to sit down and write a whole thing by hand in it, with an AI agent working alongside me. It's one of the [[topics/roadmap]] goals for the year: a genuine workflow for live-coding Sigil by hand with AI in the loop. Where does the language feel great? Where does it get in the way? We'll find out together.
- **Where we start.** From an empty file: the core game loop, a room or two, and the first commands working, then wire up the Sigil Web interface so you can watch it come alive.

## Notes

- Why does the web template have sigil-version as a dev dependency.  Why does it also only include a dev config and not a release config???
- When there is a version constraint conflict due to a transitive dependency, we should identify which package is asking for the conflicting version so that it can be corrected
- Why the hell does it take so long to build mbedtls
- Why can't we just run the code directly in sigil CLI
- How to get sigil-nrepl running inside of the game
- When eval'ing files, the runtime-compiled cached version is not being invalidated correctly.  Needs to use the same cache key logic as the store backend.  (Confirmed: the stale bytecode lived in `~/.cache/sigil/lib`; clearing it was the only way to pick up source edits.  This masked every fix and made `chain` look permanently broken.)
- We need up to date examples of how to do SSE in sigil-http and also how to do (sigil web ui) SSE-event driven UI so that it's easier to follow a tutorial to make a site like this work.
- ALso the web template in sigil CLI needs to be updated
- An unhandled error while running with `sigil eval -f` exits the process with status 0 and prints nothing, so a server that fails to start just "returns to the shell" with no clue why.  Top-level errors need to surface.

## Progress Log

- Got the HTTP server responding through sigil-web routes and middleware: routes for `/`, `/world`, and `/about`, an `/events` SSE endpoint scaffolded with `http-response/sse` + `sse-morph`, composed with `with-logging`, `with-content-type`, and `with-not-found` ([`27b5074`](https://codeberg.org/FluxHarmonic/flux-world/commit/27b5074)).
- Tracked down why `sigil deps install` was failing: the published `sigil-web` v0.14.0 tag still pinned `sigil-sxml ^0.9.0`, which collided with the `sigil-sxml 0.14` the rest of the tree wanted. Cut and pushed `sigil-web` v0.14.1 with the corrected `^0.14` pins ([`3cd63e3`](https://codeberg.org/sigil/sigil-web/commit/3cd63e3)).
- Chased the "server won't stay up / `chain` won't compile" ghost all the way down: unhandled errors in `sigil eval -f` exit silently, and the real culprit was stale bytecode in `~/.cache/sigil/lib` that never got invalidated. Clearing it let the source fixes finally take, and `chain`/`->` compile and run fine.
- Spent most of the session on the toolchain, not the MUD itself, but the whole issue list above came out of it.

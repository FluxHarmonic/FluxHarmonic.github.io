# REPL-Driven MUD Hacking

In last week's stream, we attempted to start creating a web-based MUD (Multi-User Dungeon) with Sigil and Sigil's web framework `sigil-web`.  It didn't go so well!

I spent this entire week improving the whole Sigil ecosystem to make REPL-driven coding of `sigil-web` apps from Emacs better.  Let's give it a try!

## News

**Sigil, the language and runtime, shipped six releases this week, with a seventh on the way:**

- 0.17.4: native WebAssembly builds fixed, and failures that used to vanish silently now surface.
- 0.17.5: real exception handling, `dynamic-wind` cleanup on unwind, and `sigil mcp` working again.
- 0.17.6: exception and unwind correctness across goroutines, plus GC scheduling controls.
- 0.17.7: native codegen correctness, including reentrant eval and the native-to-WASM eval path.
- 0.17.8: a unified WASM dispatch ABI on both backends, and toolchain fingerprinting for the build cache.
- 0.17.9: reproducible byte-identical builds, plus the live-eval primitives that make today's REPL workflow possible.
- 0.17.10: native live-redefinition, so I can redefine a function in a running native binary and have callers pick up the change (this one might not be tagged yet as I write this).

**Packages:**

- sigil-nrepl 0.17.0: eval abort/interrupt and streaming output.
- sigil-repl 0.17.0: bumped to the 0.17 toolchain.
- sigil-audio 0.12.0: audio plays in the browser now, through WebAudio.
- sigil-web 0.16.1: live-stream and live-collection primitives, and the real-time SSE fragment push we use tonight.
- sigil-http 0.18.0: streaming, HEAD, gzip, keep-alive, and range requests.

**Enclave, the chat platform:**

- Moved to a WASM progressive web app, with channel switching 2.1x faster by rendering straight from state.

**Tooling:**

- Kiln, our own CI, now does release automation and reproducible-build validation (8 of 9 targets byte-identical).
- live-crafter rebuilt on the new sigil-web live primitives.
